# 04 - 前端项目说明

本项目前端使用 **UmiJS 4** + **React 18** + **TypeScript** 构建，是一个简单的单页应用（SPA）。

## 1. UmiJS 4 是什么？

UmiJS 是蚂蚁集团开源的 React 应用框架，特点：

- **约定式路由**：`src/pages/` 下的文件自动变成路由。
- **内置路由、请求、状态管理、构建优化**。
- **配置化**：通过 `.umirc.ts` 配置路由、代理、构建等。

## 2. 项目结构

```
frontend/
├── .umirc.ts              # Umi 配置文件
├── package.json           # 依赖和脚本
├── tsconfig.json          # TypeScript 配置
└── src/
    ├── app.ts             # 运行时配置
    ├── pages/
    │   ├── index.tsx      # 首页，对应路由 /
    │   └── user.tsx       # 用户页，对应路由 /user
    └── services/
        └── api.ts         # 请求封装
```

## 3. 路由说明

Umi 4 的约定式路由：

| 文件 | 路由 |
|------|------|
| `src/pages/index.tsx` | `/` |
| `src/pages/user.tsx` | `/user` |

这意味着你不需要手动写路由表，只要创建文件就能访问。

## 4. 请求代理配置

`.umirc.ts` 中的 `proxy` 只在**开发时**生效：

```typescript
proxy: {
  '/api': {
    target: 'http://127.0.0.1:8080/backend',
    changeOrigin: true,
  },
},
```

开发时运行 `npm run dev`，前端在 `http://localhost:8000`，访问 `/api/users` 会被代理到 `http://127.0.0.1:8080/backend/api/users`。

**生产环境不经过这个代理**，而是直接请求 Nginx 上的 `/api/users`，由 Nginx 转发到 Tomcat。

```bash
cd frontend
npm install
npm run build
```

构建产物输出到 `frontend/dist/`：

```
frontend/dist/
├── index.html
├── umi.js
├── umi.css
└── ...
```

Nginx 会把 `frontend/dist` 作为静态资源根目录：

```nginx
root /home/charles/code/nginxdemo/frontend/dist;
```

## 6. 单页应用刷新问题

因为 Umi 使用浏览器 History 路由，访问 `/user` 时浏览器会请求服务器上的 `/user` 文件。但服务器上只有 `index.html`。

所以 Nginx 需要配置：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

含义：

1. 先尝试找 `$uri` 文件。
2. 再尝试找 `$uri/` 目录。
3. 如果都没有，返回 `/index.html`，由前端路由接管。

没有这个配置，刷新 `/user` 会报 404。

## 8. 代码注释

- `src/pages/index.tsx`：首页，展示项目说明和跳转链接。
- `src/pages/user.tsx`：用户列表页，调用后端 API 增删查数据。
- `src/services/api.ts`：封装 `fetch`，统一处理 GET/POST。
