# 09 - 常见问题排查

学习 Nginx + Tomcat 时，遇到问题很正常。本章整理了一些常见错误和排查方法。

## 1. 端口冲突

### 现象

```
bind() to 0.0.0.0:8088 failed (98: Address already in use)
```

或 Tomcat 启动时报：

```
Address already in use: bind 8080
```

### 排查

```bash
ss -tlnp | grep -E '8080|8088|8005'
```

### 解决

- 结束占用端口的进程：

```bash
kill -9 <PID>
```

- 或者修改 Nginx/Tomcat 的端口配置。

## 2. 前端页面返回 500（Permission denied）

### 现象

访问 `http://127.0.0.1:8088/` 返回 500，Nginx 错误日志显示：

```
stat() "/home/charles/code/nginxdemo/frontend/dist/index.html" failed (13: Permission denied)
```

### 原因

系统 Nginx 以 `www-data` 用户运行，而 `/home/charles` 目录默认权限为 `drwxr-x---`，
导致 Nginx 无法进入用户主目录读取静态文件。

### 解决

给 `/home/charles` 添加其他人可执行权限（只影响目录进入，不影响文件读取）：

```bash
chmod 755 /home/charles
```

然后重载 Nginx：

```bash
sudo systemctl reload nginx
```

## 3. 前端页面空白或 404

### 现象

访问 `http://127.0.0.1:8088/` 空白，或刷新 `/user` 报 404。

### 排查

1. 确认 `frontend/dist/index.html` 存在。
2. 确认 Nginx 的 `root` 指向了正确的 dist 目录。
3. 确认 `location /` 中有 `try_files $uri $uri/ /index.html;`。

### 解决

重新构建前端：

```bash
cd frontend
corepack pnpm install
corepack pnpm run build
```

## 4. 后端 API 返回 502 Bad Gateway

### 现象

Nginx 返回 502。

### 排查

1. Tomcat 是否已启动？检查端口 8080：

```bash
ss -tlnp | grep 8080
```

2. WAR 包是否在 `tomcat/webapps/backend.war`？
3. 查看 Nginx 错误日志：

```bash
tail -f /var/log/nginx/nginxdemo-error.log
```

4. 直接访问 Tomcat 测试：

```bash
curl http://127.0.0.1:8080/backend/api/users/health
```

### 解决

- 如果 Tomcat 没启动，运行 `./scripts/start-local.sh`。
- 如果 WAR 没部署，运行 `./scripts/build.sh`。
- 如果直连 Tomcat 正常但 Nginx 502，检查 `proxy_pass` 地址是否正确。

## 5. 跨域错误（CORS）

### 现象

浏览器控制台报：

```
Access to fetch at 'http://...' from origin 'http://...' has been blocked by CORS policy
```

### 排查

1. 确认前端请求的是 Nginx 地址 `http://127.0.0.1:8088/api/...`，而不是直接访问 Tomcat。
2. 如果直接访问 Tomcat，确认后端 `CorsConfig` 已开启。

### 解决

最佳实践：所有前端请求都走 Nginx，利用反向代理消除跨域。生产环境不要长期开启 `allowedOrigins("*")`。

## 6. WAR 包部署后没有自动解压

### 现象

`tomcat/webapps/` 下只有 `backend.war`，没有 `backend/` 目录。

### 排查

检查 `server.xml`：

```xml
<Host ... unpackWARs="true" autoDeploy="true">
```

### 解决

确认这两个属性为 `true`，然后重启 Tomcat。

## 7. Spring Boot 启动失败

### 现象

`catalina.out` 中有启动异常，例如：

```
Error starting ApplicationContext
```

### 排查

查看详细堆栈：

```bash
tail -n 100 tomcat/logs/catalina.out
```

常见原因：

- 端口冲突
- 缺少 `SpringBootServletInitializer`
- `spring-boot-starter-tomcat` 没有设置 `provided`

## 8. Nginx 配置文件语法错误

### 现象

Nginx 无法启动。

### 排查

```bash
sudo nginx -t
```

### 解决

根据报错修改配置。常见错误：

- 路径写错
- 缺少分号
- `upstream` 块放在了 `server` 块内部（应该放在 `http` 块内、与 `server` 同级）

## 9. 日志查看命令汇总

```bash
# Nginx 访问日志
tail -f /var/log/nginx/nginxdemo-access.log

# Nginx 错误日志
tail -f /var/log/nginx/nginxdemo-error.log

# Tomcat 标准输出
tail -f tomcat/logs/catalina.out

# Tomcat 访问日志
tail -f tomcat/logs/localhost_access_log.*.txt
```
