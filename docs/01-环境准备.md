# 01 - 环境准备

本示例采用“项目内自带 Nginx/Tomcat 二进制”的方式，因此你只需要确保 **JDK、Maven、Node.js** 已经安装好即可。

## 1. 检查 JDK

本项目使用 **JDK 17**（与 Spring Boot 3.x 兼容）。

```bash
java -version
```

期望输出包含 `openjdk version "17"` 或更高版本。

如果没有安装：

```bash
sudo apt update
sudo apt install -y openjdk-17-jdk
```

## 2. 检查 Maven

```bash
mvn -version
```

期望输出包含 `Apache Maven 3.8+`。

如果没有安装：

```bash
sudo apt install -y maven
```

## 3. 检查 Node.js / npm

```bash
node -v
npm -v
```

期望 Node.js >= 18，npm >= 9。

如果没有安装：

```bash
# 使用 NodeSource 安装 Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

## 4. Nginx 和 Tomcat

- **Nginx**：已通过 `apt` 安装为系统服务，由 `systemctl` 管理。
  项目配置文件位于 `/etc/nginx/sites-available/nginxdemo`。
- **Tomcat**：以项目内便携目录形式包含，无需安装到系统。
  目录：`tomcat/apache-tomcat-10.1.56`。

## 5. 验证网络工具

`scripts/start-local.sh` 中用 `nc` 检查端口，建议安装：

```bash
sudo apt install -y netcat-openbsd
```

或者你可以把脚本里的 `nc -z 127.0.0.1 8080` 改成：

```bash
ss -tlnp | grep -q ':8080'
```

## 6. 目录权限

系统 Nginx 以 `www-data` 用户运行，需要能够读取 `/home/charles/code/nginxdemo/frontend/dist` 中的静态文件。

如果访问首页返回 500，请检查 `/home/charles` 是否对其他人开放了执行权限：

```bash
ls -ld /home/charles
# 应该是 drwxr-xr-x，如果不是，执行：
chmod 755 /home/charles
```

## 7. 端口占用检查

启动前请确保以下端口没有被占用：

| 端口 | 用途 |
|------|------|
| 8080 | Tomcat |
| 8088 | Nginx |
| 8005 | Tomcat shutdown |

检查命令：

```bash
ss -tlnp | grep -E '8080|8088|8005'
```

## 7. WSL 下的 npm 注意事项

如果你在 WSL 中运行，而 `which npm` 指向的是 Windows 版本的 npm（例如 `/mnt/c/Program Files/nodejs/npm`），直接运行 `npm install` 可能会因为 UNC 路径问题失败。

本项目的一键构建脚本 `scripts/build.sh` 会自动检测并使用 `corepack pnpm`（Node.js 内置的包管理器）来安装前端依赖，避免这个问题。

如果你希望手动安装，可以使用：

```bash
cd frontend
corepack pnpm install
corepack pnpm run build
```

## 8. 目录权限

所有文件都在 `/home/charles/code/nginxdemo` 下，确保当前用户有读写权限即可。

```bash
ls -la /home/charles/code/nginxdemo
```

如果没有问题，就可以进入下一步：构建并启动。
