# 05 - 后端项目说明

本项目后端使用 **Spring Boot 3.2.x** + **Java 17**，并被打包成 **WAR** 格式，部署到外置 Tomcat。

## 1. 项目结构

```
backend/
├── pom.xml
└── src/
    ├── main/
    │   ├── java/com/example/demo/
    │   │   ├── DemoApplication.java          # 启动类
    │   │   ├── controller/
    │   │   │   └── UserController.java       # REST API
    │   │   ├── service/
    │   │   │   └── UserService.java          # 业务逻辑
    │   │   └── config/
    │   │       └── CorsConfig.java           # 跨域配置
    │   └── resources/
    │       └── application.yml               # 应用配置
    └── test/
```

## 2. 关键：pom.xml 中 WAR 打包

```xml
<packaging>war</packaging>
```

这告诉 Maven 最终生成 `backend.war` 而不是 `backend.jar`。

## 3. 关键：spring-boot-starter-tomcat 的 scope

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-tomcat</artifactId>
    <scope>provided</scope>
</dependency>
```

- `scope=provided` 表示：**编译时需要，但运行时不打包**。
- 因为外置 Tomcat 已经提供了这些类，如果打包进去会造成冲突或体积变大。

## 4. 关键：继承 SpringBootServletInitializer

```java
@SpringBootApplication
public class DemoApplication extends SpringBootServletInitializer {

    @Override
    protected SpringApplicationBuilder configure(SpringApplicationBuilder application) {
        return application.sources(DemoApplication.class);
    }

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

- 外置 Tomcat 启动 WAR 时，会先调用 `SpringBootServletInitializer.configure()`。
- 这里必须返回主类 `DemoApplication.class`，否则 Spring Boot 无法初始化。
- `main` 方法保留，用于本地开发时直接运行（内嵌 Tomcat）。

## 5. application.yml 配置

```yaml
server:
  port: 8081
  servlet:
    context-path: /backend
```

- `server.port`：内嵌 Tomcat 开发端口。
- `server.servlet.context-path`：应用的上下文路径。

外置 Tomcat 部署时，`server.port` 会被忽略，但 `context-path` 仍然有效。不过通常 WAR 包名决定上下文路径，`backend.war` 对应 `/backend`。

## 6. REST API 说明

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/users` | 获取所有用户 |
| GET | `/api/users/{id}` | 根据 ID 获取用户 |
| POST | `/api/users` | 新增用户 |
| GET | `/api/users/health` | 健康检查 |

部署到 Tomcat 后，完整地址为：

```
http://localhost:8080/backend/api/users
```

经过 Nginx 反向代理后，前端访问：

```
http://localhost:8088/api/users
```

## 7. 跨域配置

虽然 Nginx 反向代理后已经同源，但我们仍然提供了 CORS 配置：

```java
@Configuration
public class CorsConfig {
    @Bean
    public WebMvcConfigurer corsConfigurer() {
        return new WebMvcConfigurer() {
            @Override
            public void addCorsMappings(CorsRegistry registry) {
                registry.addMapping("/api/**")
                        .allowedOrigins("*")
                        .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS");
            }
        };
    }
}
```

> 生产环境不要把 `allowedOrigins` 写成 `*`。

## 8. 常用命令

```bash
cd backend

# 本地开发运行（内嵌 Tomcat）
mvn spring-boot:run

# 打包成 WAR
mvn clean package -DskipTests

# 运行 WAR（也会启动内嵌 Tomcat）
java -jar target/backend.war
```
