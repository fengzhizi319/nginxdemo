# 08 - 负载均衡演示

Nginx 的 `upstream` 模块可以把请求分发到多台后端服务器，这就是**负载均衡**。

## 1. 当前配置

`nginx/conf.d/default.conf`：

```nginx
upstream backend_servers {
    server 127.0.0.1:8080 weight=1;
    # server 127.0.0.1:8082 weight=1;
}
```

目前只有一台 Tomcat 在 8080 端口。

## 2. 添加第二个后端实例

### 2.1 复制 Tomcat 运行目录

```bash
cd /home/charles/code/nginxdemo

# 复制一份 CATALINA_BASE
cp -r tomcat tomcat-8082
```

### 2.2 修改新实例的端口

编辑 `tomcat-8082/conf/server.xml`：

```xml
<Server port="8006" shutdown="SHUTDOWN">
  <Connector port="8082" protocol="HTTP/1.1" />
```

- `Server port` 从 8005 改为 8006（避免冲突）。
- `Connector port` 从 8080 改为 8082。

### 2.3 准备 WAR 包

```bash
cp tomcat/webapps/backend.war tomcat-8082/webapps/
```

### 2.4 启动第二个实例

```bash
export CATALINA_HOME=/home/charles/code/nginxdemo/tomcat/apache-tomcat-10.1.56
export CATALINA_BASE=/home/charles/code/nginxdemo/tomcat-8082
$CATALINA_HOME/bin/startup.sh
```

### 2.5 修改 Nginx upstream

```nginx
upstream backend_servers {
    server 127.0.0.1:8080 weight=1;
    server 127.0.0.1:8082 weight=1;
}
```

然后重载 Nginx 配置：

```bash
sudo systemctl reload nginx
# 或 ./scripts/reload-nginx.sh
```

## 3. 验证负载均衡

连续请求：

```bash
for i in {1..10}; do
    curl -s http://127.0.0.1:8088/api/users/health
    echo ""
done
```

为了区分两个后端，你可以在 `UserController.health()` 中返回当前实例的端口号：

```java
@GetMapping("/health")
public Map<String, String> health() {
    return Map.of(
        "status", "UP",
        "port", environment.getProperty("server.port", "unknown"),
        "message", "后端服务运行正常"
    );
}
```

这样你会看到响应在 8080 和 8082 之间交替。

## 4. 负载均衡策略

Nginx 支持多种策略：

| 策略 | 说明 | 配置示例 |
|------|------|---------|
| 轮询（默认） | 请求依次分发 | 默认即可 |
| weight | 按权重分发 | `server 127.0.0.1:8080 weight=3;` |
| ip_hash | 按客户端 IP 哈希，保持会话 | `ip_hash;` |
| least_conn | 最少连接数优先 | `least_conn;` |
| backup | 备用服务器 | `server 127.0.0.1:8082 backup;` |

示例：

```nginx
upstream backend_servers {
    server 127.0.0.1:8080 weight=3;
    server 127.0.0.1:8082 weight=1;
    server 127.0.0.1:8084 backup;
}
```

## 5. 健康检查（高级）

Nginx 开源版的 `upstream` 只提供简单的失败重试，没有主动健康检查模块。

如果需要主动健康检查，可以使用：

- Nginx Plus（商业版）。
- 第三方模块 `nginx_upstream_check_module`。
- 在本示例中，可以通过 `max_fails` 和 `fail_timeout` 做基础容错：

```nginx
upstream backend_servers {
    server 127.0.0.1:8080 max_fails=3 fail_timeout=30s;
    server 127.0.0.1:8082 max_fails=3 fail_timeout=30s;
}
```

含义：如果某台服务器在 30 秒内失败 3 次，Nginx 会暂时把它标记为不可用。
